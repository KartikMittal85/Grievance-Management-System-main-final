About VoiceUp — Grievance Management System

VoiceUp is an intuitive, secure platform built to collect, track and resolve student grievances efficiently. It brings transparency to the grievance redressal workflow by providing students, admins and VIP staff a single, auditable place to raise and manage issues.

Project Overview

VoiceUp allows students to submit grievances (academic, hostel, examination, library, etc.) and enables administrators to triage, update, and resolve them. Each grievance has a status and optional remarks from admins. The system also supports feedback submission and basic analytics.

Key Features
Submit Grievance

Students can submit details and attachments; timestamped and stored.

Admin Management

Admins can view, update status, add remarks, and resolve or reject grievances.

VIP Panel

VIP users get a themed panel for priority handling and quick views.

Feedback Module

Students provide UI feedback; admins can review responses.

Audit Trail

Status updates are saved with admin remarks for traceability.

Responsive UI

Built with Bootstrap for mobile-friendly interaction.

Tech Stack
  • Backend: Flask (Python), SQLAlchemy
  • Frontend: HTML, Bootstrap, jQuery, Jinja templates
  • Database: SQLite
  • Auth: Flask-Login + password hashing (werkzeug.security)
  • Hosting (optional): Heroku / PythonAnywhere / VPS
How It Works — Workflow
1
Student registers / logs in.
Student provides credentials and lands on the dashboard.
2
Student submits a grievance.
Title, category, description and optional tags are stored with a default status.
3
Admin reviews and updates status.
Admin changes status (Seen → In Progress → Resolved / Rejected) and adds remarks.
4
Student views updates and feedback.
Student can view remarks, final decision, and provide feedback.
